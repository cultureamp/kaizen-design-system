# Layout and Spacing

<br/>

## Naming convention

Kaizen's spacing tokens use pixel values to reference equivalent rem values. For example, the spacing token `16` refers to `1rem`, because `16px` = `1rem`.

<SpacingTokens />

<br/>

## Reasoning

Engineers need to use `rem` when spacing for accessibility reasons. However, Figma does not allow `rem` values when creating designs. This approach allows engineers to recieve pixel values from a designer, and write code in terms of pixel values without having to calculate what the corresponding `rem` value would be.

<br/>

## Implementation

Kaizen's spacing tokens are easily available through our [Tailwind package](/story/systems-tailwind-overview--page).

For example, if a design specifies a `div` with a margin-right of `12px`, the spacing token `12` can be applied to Tailwind's `mr-` utility class.

```tsx
<div className="flex">
  <div className="mr-12">   // This will become margin-right: 0.75rem
    <Button label="Button 1" />
  </div>
  <Button label="Button 2" />
</div>
```
