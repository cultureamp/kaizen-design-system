`status` updates the styling of the validation FieldMessage.
`validationMessage` is descriptive message for the 'status' states.

A `validationMessage` must be provided in order for the validation status to be rendered

### `trigger`
Replaces the trigger button

Exposes the trigger properties and the ref to be used on the replacing trigger 

```tsx
<Select
  id="select-dropdown-selected"
  label="label"
  items={singleMockItems}
  trigger={(): JSX.Element => (
    <CustomTriggerButton
      selectedKeys={selectedKeys}
      label={args.label}
    />
  )}
/>
```

### `children`
Replaces the contents of the Listbox and describes how the options are displayed.

Exposes the option properties which contains the items

```tsx
<Select
  id="select-custom-option"
  label="label"
  items={singleMockItems}
  >
  {(optionsProps): JSX.Element[] =>
      optionsProps.items.map((item: Node<SingleItemType>) => (
      <CustomOption
          {...optionsProps}
          key={item.key}
          item={item}
      />
      ))
  }
</Select>
```

### `onSelectionChange`
Handler that is called when the selection changes.

```tsx
const [selected, setSelected] = useState("id-sre")

<Select
  label="label"
  id="single-select"
  items={singleMockItems}
  selectedKey={selected}
  onSelectionChange={(key):void => setSelected(key)}
/>
```
