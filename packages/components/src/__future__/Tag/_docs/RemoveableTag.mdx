## Overview

Similar to the Tag component, but with a clear icon, that recieves a callback from props.
The consumer should manage the state regarding whether or not the icon should be rendered,
and pass a callback to remove the RemoveableTag component via the `removeButtonProps` prop.

## API

<Controls of={RemoveableTagStories.Playground} />

### `children`

Takes a `ReactNode`. This should be the text that is displayed inside the Tag.

<Canvas of={RemoveableTagStories.Children} />

### `color`

Determines the color of the background, text, and icon.

<Canvas of={RemoveableTagStories.Color} />

### `icon`

An optional icon can be passed via this prop. The icon will be rendered on the left of `children`
in LTR, and on the right of `children` in RTL. For a list of icons, see [Icons](?path=/story/components-icons--sticker-sheet-default)

<Canvas of={RemoveableTagStories.Icon} />

### `classNameOverride`

Allows classnames to be attached to the outermost element of Tag.

<Canvas of={RemoveableTagStories.ClassNameOverride} />

### `removeButtonProps`

An object containing the following properties that relate to the clear button:

#### `aria-label`

An aria label that is attached to the clear button.

#### `onClick`

A callback that gets invoked when clicking the clear button.
